## 1. Ôn lại: Bagging vs Boosting — Hai triết lý khác nhau

Ở Bài 19, Bagging giải quyết vấn đề **variance cao** của Decision Tree bằng cách xây nhiều cây **song song**, độc lập nhau, rồi lấy đa số phiếu.

**Boosting** tiếp cận hoàn toàn khác: xây các cây **tuần tự**, mỗi cây sau **tập trung sửa lỗi** của cây trước.

**Ẩn dụ để hiểu sự khác biệt:**

```
Bagging:   Hỏi 100 chuyên gia CÙNG LÚC → lấy ý kiến đa số
           (mỗi người làm việc độc lập)

Boosting:  Hỏi chuyên gia 1 → ghi nhận chỗ sai
           → Chuyên gia 2 tập trung vào chỗ sai đó → ghi nhận chỗ sai mới
           → Chuyên gia 3 tập trung vào chỗ sai mới ...
           → Kết hợp tất cả theo trọng số
```

**Điểm khác biệt cốt lõi:**

| | Bagging / Random Forest | Boosting |
|---|---|---|
| Xây dựng cây | Song song, độc lập | Tuần tự, phụ thuộc nhau |
| Mục tiêu | Giảm **Variance** | Giảm **Bias** |
| Cây thành phần | Cây sâu (full tree) | Cây nông (weak learner) |
| Dễ overfit | Khó | Có thể nếu tuning sai |
| Song song hóa | ✅ Dễ | ❌ Khó (tuần tự) |
| Hiệu năng thực tế | Tốt | **Thường tốt hơn** |

## 2. AdaBoost — Adaptive Boosting

### 2.1. Ý tưởng

AdaBoost (Freund & Schapire, 1997) là thuật toán Boosting đầu tiên được sử dụng rộng rãi. Ý tưởng đơn giản: **tăng trọng số** cho những quan sát bị dự đoán sai, để mô hình tiếp theo chú ý nhiều hơn vào chúng.

**Hình dung:**

```
Vòng 1: Học trên dữ liệu gốc (tất cả quan sát có trọng số bằng nhau)
        → Dự đoán sai quan sát {3, 7, 12}

Vòng 2: Tăng trọng số cho {3, 7, 12}
        → Học trên dữ liệu có trọng số mới
        → Dự đoán sai quan sát {7, 15, 21}

Vòng 3: Tăng trọng số cho {7, 15, 21}
        → Học trên dữ liệu có trọng số mới
        ...

Kết quả: Kết hợp tất cả mô hình theo trọng số α
         (mô hình đúng nhiều → α cao → ảnh hưởng nhiều hơn)
```

### 2.2. Thuật toán AdaBoost (cho phân loại nhị phân)

**Đầu vào:** Tập huấn luyện $\{(x_i, y_i)\}_{i=1}^N$, với $y_i \in \{-1, +1\}$, số vòng lặp $M$.

**Khởi tạo:** Trọng số đều nhau cho tất cả quan sát:

$$w_i^{(1)} = \frac{1}{N}, \quad i = 1, \ldots, N$$

**Lặp** $m = 1, 2, \ldots, M$:

1. Huấn luyện weak learner $h_m(x)$ trên dữ liệu có trọng số $\{w_i^{(m)}\}$.

2. Tính weighted error (lỗi có trọng số):

$$\epsilon_m = \sum_{i=1}^{N} w_i^{(m)} \cdot \mathbf{1}[h_m(x_i) \neq y_i]$$

3. Tính trọng số của mô hình $m$ (mô hình đúng nhiều → $\alpha$ lớn):

$$\alpha_m = \frac{1}{2} \ln\left(\frac{1 - \epsilon_m}{\epsilon_m}\right)$$

4. Cập nhật trọng số quan sát (sai → tăng, đúng → giảm):

$$w_i^{(m+1)} = w_i^{(m)} \cdot \exp\left(-\alpha_m \cdot y_i \cdot h_m(x_i)\right)$$

Chuẩn hóa lại để tổng trọng số = 1.

**Dự đoán cuối cùng:**

$$H(x) = \text{sign}\left(\sum_{m=1}^{M} \alpha_m \cdot h_m(x)\right)$$

**Ví dụ trực quan với $\epsilon_m$:**

| $\epsilon_m$ | $\alpha_m$ | Ý nghĩa |
|---|---|---|
| 0.1 (rất tốt) | 1.10 | Mô hình được tin tưởng nhiều |
| 0.3 (tốt) | 0.42 | Mô hình được tin tưởng vừa |
| 0.5 (random) | 0.00 | Mô hình không đóng góp gì |
| > 0.5 (tệ) | < 0 | Mô hình bị đảo ngược |

## 3. Gradient Boosting (GBM)

### 3.1. Từ AdaBoost đến Gradient Boosting

AdaBoost hoạt động tốt nhưng bị giới hạn ở bài toán phân loại nhị phân. **Gradient Boosting** (Friedman, 2001) tổng quát hóa ý tưởng Boosting bằng ngôn ngữ của **Gradient Descent** — có thể áp dụng cho bất kỳ hàm mất mát nào.

**Ý tưởng cốt lõi:** Thay vì tăng trọng số quan sát sai (như AdaBoost), Gradient Boosting xây mỗi cây mới để **dự đoán phần sai sót (residual)** của mô hình hiện tại.

**Hình dung cho hồi quy:**

```
Mô hình ban đầu: F₀(x) = ȳ (trung bình)

Vòng 1:
  Residual₁ = y - F₀(x)        ← phần chưa giải thích được
  Cây₁ học để dự đoán Residual₁
  F₁(x) = F₀(x) + η × Cây₁(x) ← cập nhật mô hình (η = learning rate)

Vòng 2:
  Residual₂ = y - F₁(x)        ← residual nhỏ hơn rồi
  Cây₂ học để dự đoán Residual₂
  F₂(x) = F₁(x) + η × Cây₂(x)

...tiếp tục M vòng...

Kết quả: F_M(x) = F₀(x) + η×Cây₁(x) + η×Cây₂(x) + ... + η×Cây_M(x)
```

### 3.2. Công thức tổng quát

Tại mỗi bước $m$, ta tính **pseudo-residual** (gradient âm của loss function):

$$r_i^{(m)} = -\left[\frac{\partial L(y_i, F(x_i))}{\partial F(x_i)}\right]_{F=F_{m-1}}$$

Huấn luyện cây $h_m$ để dự đoán $r_i^{(m)}$, rồi cập nhật:

$$F_m(x) = F_{m-1}(x) + \eta \cdot h_m(x)$$

**Với các hàm loss khác nhau, pseudo-residual có dạng khác:**

| Bài toán | Loss Function | Pseudo-residual |
|---|---|---|
| Hồi quy | MSE: $\frac{1}{2}(y-F)^2$ | $y - F(x)$ (residual thông thường) |
| Phân loại | Log-loss | $y - p(x)$ (residual xác suất) |
| Robust | Absolute loss | $\text{sign}(y - F(x))$ |

### 3.3. Siêu tham số của GBM

| Tham số | Ý nghĩa | Ảnh hưởng |
|---|---|---|
| `n.trees` / `nrounds` | Số vòng lặp (số cây) | Nhiều → có thể overfit |
| `shrinkage` / `eta` | Learning rate $\eta$ | Nhỏ → chậm nhưng ổn định hơn |
| `interaction.depth` | Độ sâu mỗi cây | Sâu → phức tạp hơn |
| `n.minobsinnode` | Số obs tối thiểu tại leaf | Lớn → regularization |
| `bag.fraction` | Tỷ lệ quan sát lấy mẫu mỗi vòng | < 1 → Stochastic GBM |
| `subsample` | Như bag.fraction (XGBoost) | < 1 → giảm variance |

> 💡 **Quy tắc ngón tay cái:**
> Learning rate nhỏ ($\eta \leq 0.1$) + nhiều cây → thường tốt hơn learning rate lớn + ít cây.
> Dùng **Early Stopping** để tìm số cây tối ưu tự động.

## 4. XGBoost & LightGBM — Phiên bản tối ưu hóa

### 4.1. XGBoost (eXtreme Gradient Boosting)

XGBoost (Chen & Guestrin, 2016) là cải tiến của GBM với nhiều tối ưu hóa quan trọng:

**Về thuật toán:**
- Thêm **regularization** (L1 và L2) vào hàm mục tiêu để tránh overfitting.
- Dùng **second-order Taylor expansion** thay vì chỉ gradient bậc nhất → hội tụ nhanh hơn.

**Về kỹ thuật:**
- Xử lý **missing values** tự động (tìm hướng phân chia tốt nhất cho NA).
- Hỗ trợ **parallel processing** (dù các cây vẫn tuần tự, việc tìm điểm phân chia tối ưu được song song hóa).
- **Early stopping**: dừng khi validation metric không cải thiện sau K vòng.

**Hàm mục tiêu của XGBoost:**

$$\mathcal{L} = \underbrace{\sum_{i=1}^{N} L(y_i, \hat{y}_i)}_{\text{training loss}} + \underbrace{\sum_{m=1}^{M} \Omega(h_m)}_{\text{regularization}}$$

trong đó $\Omega(h) = \gamma T + \frac{1}{2}\lambda \|w\|^2$ ($T$ = số lá, $w$ = giá trị tại lá).

### 4.2. LightGBM

LightGBM (Microsoft, 2017) tiếp tục tối ưu hóa tốc độ với hai kỹ thuật chính:

- **GOSS (Gradient-based One-Side Sampling):** Giữ lại các quan sát có gradient lớn (đang học kém), lấy mẫu ngẫu nhiên các quan sát có gradient nhỏ → giảm dữ liệu cần xử lý mà không mất nhiều thông tin.

- **EFB (Exclusive Feature Bundling):** Gộp các biến thưa (sparse features) hiếm khi khác 0 cùng lúc thành một biến → giảm số biến cần xét.

- **Leaf-wise tree growth** (thay vì level-wise như XGBoost): Chọn leaf có gain lớn nhất để phân chia tiếp → cây không đều nhưng hội tụ nhanh hơn.

### 4.3. So sánh GBM, XGBoost, LightGBM

| | GBM (`gbm`) | XGBoost | LightGBM |
|---|---|---|---|
| Tốc độ | Chậm | Nhanh | **Nhanh nhất** |
| Bộ nhớ | Cao | Trung bình | **Thấp nhất** |
| Regularization | Giới hạn | ✅ L1, L2 | ✅ L1, L2 |
| Missing values | Cần xử lý thủ công | ✅ Tự động | ✅ Tự động |
| Categorical features | Cần encode | Cần encode | ✅ Hỗ trợ trực tiếp |
| Early stopping | ✅ | ✅ | ✅ |
| Phổ biến trong thực tế | Trung bình | **Rất cao** | Cao |

```r
german <- read.csv("german1.csv")
german$target <- factor(german$target, levels = c(1, 2),
```

**Quan sát trọng số thay đổi theo vòng lặp (minh họa thủ công):**

```r
library(rpart)

# Dùng biến số để đơn giản
g_num <- german %>%
  select(where(is.numeric), target) %>%
  mutate(y = ifelse(target == "Bad", 1, -1)) %>%
  select(-target)

n <- nrow(g_num)
w <- rep(1/n, n)  # Trọng số ban đầu bằng nhau

set.seed(42)
M_demo <- 5  # Demo 5 vòng

for (m in 1:M_demo) {
  # Huấn luyện cây nông (decision stump: depth = 1)
  tree_m <- rpart(y ~ . - y, data = g_num, method = "anova",
                  weights = w,
                  control = rpart.control(maxdepth = 1, cp = 0))

  pred_m  <- sign(predict(tree_m, g_num))
  correct <- (pred_m == g_num$y)

  # Weighted error
  eps_m   <- sum(w[!correct]) / sum(w)
  alpha_m <- 0.5 * log((1 - eps_m) / eps_m)

  # Cập nhật trọng số
  w <- w * exp(-alpha_m * g_num$y * pred_m)
  w <- w / sum(w)  # Chuẩn hóa

  cat(sprintf("Vòng %d | eps=%.4f | alpha=%.4f | w_max=%.4f | w_min=%.6f\n",
              m, eps_m, alpha_m, max(w), min(w)))
}
```

> 💡 **Quan sát:** `w_max` tăng dần theo vòng lặp — những quan sát khó (bị sai nhiều lần) nhận trọng số ngày càng lớn, buộc các mô hình sau phải chú ý nhiều hơn.

### 5.2. Gradient Boosting với `gbm` — Titanic

```r
titanic <- read.csv("titanic.csv", stringsAsFactors = TRUE)

titanic_clean <- titanic %>%
  select(survived, pclass, sex, age, sibsp, parch, fare) %>%
  mutate(
    survived = as.integer(survived),       # gbm cần 0/1 numeric
    sex      = as.integer(sex == "female")
  ) %>%
  drop_na()

# Chia train/test bằng caret
set.seed(42)
tit_idx   <- createDataPartition(titanic_clean$survived, p = 0.8, list = FALSE)
tit_train <- titanic_clean[tit_idx, ]
tit_test  <- titanic_clean[-tit_idx, ]

# Huấn luyện GBM
set.seed(42)
gbm_model <- gbm(
  survived ~ .,
  data              = tit_train,
  distribution      = "bernoulli",  # Phân loại nhị phân
  n.trees           = 500,
  shrinkage         = 0.05,         # Learning rate η
  interaction.depth = 3,            # Độ sâu mỗi cây
  n.minobsinnode    = 10,
  bag.fraction      = 0.8,          # Stochastic GBM
  cv.folds          = 5,            # Cross-validation để tìm ntree tối ưu
  verbose           = FALSE
)

# Tìm số cây tối ưu qua CV
best_ntree <- gbm.perf(gbm_model, method = "cv", plot.it = TRUE)
cat("Số cây tối ưu (CV):", best_ntree, "\n")

# Dự đoán với số cây tối ưu
pred_gbm <- predict(gbm_model, newdata = tit_test,
                    n.trees = best_ntree, type = "response")

roc_gbm <- roc(tit_test$survived, pred_gbm, quiet = TRUE)
cat("AUC (GBM):", round(auc(roc_gbm), 4), "\n")
```

### 5.3. XGBoost — German Credit

XGBoost yêu cầu dữ liệu ở dạng **matrix số**, không dùng được data.frame với biến phân loại trực tiếp.

```r
# Chuẩn bị dữ liệu: chuyển sang matrix số
X_train <- model.matrix(target ~ . - 1, data = g_train)
X_test  <- model.matrix(target ~ . - 1, data = g_test)

y_train <- as.integer(g_train$target == "Bad")  # 1 = Bad, 0 = Good
y_test  <- as.integer(g_test$target  == "Bad")

# Tạo DMatrix — định dạng tối ưu của XGBoost
dtrain <- xgb.DMatrix(data = X_train, label = y_train)
dtest  <- xgb.DMatrix(data = X_test,  label = y_test)

# Tham số XGBoost
params <- list(
  objective        = "binary:logistic",
  eval_metric      = "auc",
  eta              = 0.05,
  max_depth        = 4,
  subsample        = 0.8,
  colsample_bytree = 0.8,
  lambda           = 1,    # L2 regularization
  alpha            = 0     # L1 regularization
)

# Huấn luyện với Early Stopping
set.seed(42)
xgb_model <- xgb.train(
  params                = params,
  data                  = dtrain,
  nrounds               = 500,
  watchlist             = list(train = dtrain, test = dtest),
  early_stopping_rounds = 30,
  print_every_n         = 50,
  verbose               = 1
)

cat("\nSố vòng tối ưu (Early Stopping):", xgb_model$best_iteration, "\n")
cat("AUC tốt nhất (validation):      ", round(xgb_model$best_score, 4), "\n")
```

**Đánh giá trên test set:**

```r
pred_xgb <- predict(xgb_model, newdata = dtest)
```

### 5.4. XGBoost — Wines (Classification)

```r
wines <- read.csv("wines.csv", stringsAsFactors = TRUE)
wines$type <- factor(wines$type)

# Chia train/test bằng caret
set.seed(42)
w_idx   <- createDataPartition(wines$type, p = 0.8, list = FALSE)
w_train <- wines[w_idx, ]
w_test  <- wines[-w_idx, ]

# Chuẩn bị matrix cho XGBoost (bỏ quality, chỉ dùng biến hóa học)
X_w_train <- model.matrix(type ~ . - quality - 1, data = w_train)
X_w_test  <- model.matrix(type ~ . - quality - 1, data = w_test)

y_w_train <- as.integer(w_train$type == "red")
y_w_test  <- as.integer(w_test$type  == "red")

dw_train <- xgb.DMatrix(data = X_w_train, label = y_w_train)
dw_test  <- xgb.DMatrix(data = X_w_test,  label = y_w_test)

# Tuning XGBoost với caret
ctrl_w <- trainControl(
  method          = "cv",
  number          = 5,
  classProbs      = TRUE,
  summaryFunction = twoClassSummary,
  verboseIter     = FALSE
)

xgb_grid_w <- expand.grid(
  nrounds          = c(100, 200),
  eta              = c(0.05, 0.1),
  max_depth        = c(3, 4, 6),
  gamma            = 0,
  colsample_bytree = 0.8,
  min_child_weight = 1,
```

### 5.5. Early Stopping — Tránh Overfitting

Early Stopping là kỹ thuật quan trọng nhất để kiểm soát overfitting trong Boosting.

```r
# Minh họa overfitting với GBM khi không có early stopping
set.seed(42)
gbm_overfit <- gbm(
  survived ~ .,
  data              = tit_train,
  distribution      = "bernoulli",
  n.trees           = 1000,
  shrinkage         = 0.1,
  interaction.depth = 5,
  bag.fraction      = 1.0,   # Không stochastic
  cv.folds          = 0,     # Không CV
  verbose           = FALSE
)

# Tính AUC train và test theo từng số cây
ntree_seq   <- seq(10, 1000, by = 10)
auc_train_v <- numeric(length(ntree_seq))
auc_test_v  <- numeric(length(ntree_seq))

for (i in seq_along(ntree_seq)) {
  p_tr <- predict(gbm_overfit, tit_train,
                  n.trees = ntree_seq[i], type = "response")
  p_te <- predict(gbm_overfit, tit_test,
                  n.trees = ntree_seq[i], type = "response")

  auc_train_v[i] <- auc(roc(tit_train$survived, p_tr, quiet = TRUE))
  auc_test_v[i]  <- auc(roc(tit_test$survived,  p_te, quiet = TRUE))
}

# Vẽ đồ thị
data.frame(ntree = ntree_seq,
           Train = auc_train_v,
           Test  = auc_test_v) %>%
  pivot_longer(-ntree, names_to = "Set", values_to = "AUC") %>%
  ggplot(aes(x = ntree, y = AUC, color = Set)) +
  geom_line(linewidth = 0.9) +
  geom_vline(xintercept = ntree_seq[which.max(auc_test_v)],
             linetype = "dashed", color = "gray40") +
  scale_color_manual(values = c("Train" = "steelblue", "Test" = "tomato")) +
  labs(title    = "Overfitting trong GBM — Titanic",
       subtitle  = "Đường đứt: số cây tối ưu cho test AUC",
       x = "Số cây (n.trees)", y = "AUC") +
  theme_minimal()

best_ntree_manual <- ntree_seq[which.max(auc_test_v)]
cat("Số cây cho test AUC cao nhất:", best_ntree_manual, "\n")
cat("Train AUC tại điểm đó:       ",
    round(auc_train_v[which.max(auc_test_v)], 4), "\n")
cat("Test AUC cao nhất:            ", round(max(auc_test_v), 4), "\n")
```

> ⚠️ **Nhận xét:** Train AUC liên tục tăng đến 1.0, trong khi Test AUC tăng rồi giảm — đây là dấu hiệu overfitting rõ ràng. Early Stopping giải quyết vấn đề này bằng cách dừng đúng lúc.

## 6. Hướng dẫn Tuning thực tế

Tuning Boosting có nhiều tham số hơn Random Forest. Dưới đây là **chiến lược theo bước** cho người mới:

### Bước 1: Cố định learning rate thấp, tìm số cây

```r
# Dùng learning rate = 0.05, tìm n.trees qua CV
# Cần target dạng numeric 0/1 cho gbm
g_train_num <- g_train %>%
  mutate(target_num = as.integer(target == "Bad")) %>%
  select(-target)

set.seed(42)
gbm_step1 <- gbm(target_num ~ ., data = g_train_num,
                  distribution = "bernoulli",
                  n.trees = 1000, shrinkage = 0.05,
                  interaction.depth = 4, cv.folds = 5,
                  verbose = FALSE)
best_n <- gbm.perf(gbm_step1, method = "cv")
cat("Số cây tối ưu với eta=0.05:", best_n, "\n")
```

### Bước 2: Tune depth và min obs

```r
ctrl_step2 <- trainControl(method = "cv", number = 5,
                           classProbs = TRUE,
                           summaryFunction = twoClassSummary)
grid_step2 <- expand.grid(
  n.trees           = best_n,               # Cố định từ bước 1
  interaction.depth = c(2, 3, 4, 5),
  shrinkage         = 0.05,
  n.minobsinnode    = c(5, 10, 20)
)

set.seed(42)
m_step2 <- train(target ~ ., g_train, method = "gbm",
                 trControl = ctrl_step2, metric = "ROC",
                 tuneGrid = grid_step2, verbose = FALSE)
print(m_step2$bestTune)
```

### Bước 3: Giảm learning rate, tăng số cây tương ứng

```r
# Nếu eta giảm 2x → n.trees tăng 2x để giữ nguyên "tổng bước học"
# eta = 0.05, n.trees = 200 ≈ eta = 0.025, n.trees = 400
```

### Bước 4: Thêm stochastic sampling

```r
grid_step4 <- expand.grid(
  n.trees           = best_n * 2,
  interaction.depth = m_step2$bestTune$interaction.depth,
  shrinkage         = 0.025,
  n.minobsinnode    = m_step2$bestTune$n.minobsinnode
)
# bag.fraction chỉ có thể truyền qua argument trực tiếp trong gbm(),
# không qua tuneGrid của caret — cần huấn luyện trực tiếp:
set.seed(42)
gbm_step4 <- gbm(target_num ~ ., data = g_train_num,
                  distribution = "bernoulli",
                  n.trees           = best_n * 2,
                  shrinkage         = 0.025,
                  interaction.depth = m_step2$bestTune$interaction.depth,
                  n.minobsinnode    = m_step2$bestTune$n.minobsinnode,
                  bag.fraction      = 0.8,
                  cv.folds          = 5,
                  verbose           = FALSE)
best_n4 <- gbm.perf(gbm_step4, method = "cv")
cat("Số cây tối ưu (bước 4):", best_n4, "\n")
```
